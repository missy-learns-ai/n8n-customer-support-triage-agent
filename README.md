# n8n Customer Support Triage Agent

This repository contains a two-workflow n8n template for AI-assisted customer support triage with Gmail, Google Sheets, Slack, and OpenAI.

The template watches incoming support emails, classifies and drafts replies with AI, logs each ticket to Google Sheets, and sends Slack approval buttons so a human can approve, edit, or reject the suggested response.

## Workflows

| File | Purpose |
| --- | --- |
| `workflows/Customer Support Triage Agent.json` | Main triage workflow. It reads support emails from Gmail, extracts ticket fields, classifies the request with OpenAI, writes the ticket to Google Sheets, and posts a Slack review message. |
| `workflows/Support Approval Button Handler.json` | Slack approval handler. It receives Slack button actions, looks up the related ticket, updates Google Sheets, updates the Slack message, and sends the approved reply through Gmail. |

## What It Does

1. Watches Gmail for messages with a support label.
2. Extracts the sender, subject, message body, thread ID, and message ID.
3. Uses OpenAI to classify category, priority, sentiment, intent, summary, confidence, and a suggested reply.
4. Detects whether the message is a new ticket or part of an existing thread.
5. Logs ticket data into Google Sheets.
6. Sends a Slack message with action buttons:
   - Approve AI Reply
   - Needs Edit
   - Reject
7. Handles Slack button responses in a second workflow.
8. Sends the approved Gmail reply and updates the ticket status.

## Requirements

- n8n Cloud or self-hosted n8n
- Gmail account with OAuth credentials connected in n8n
- Google Sheets account with OAuth credentials connected in n8n
- OpenAI API credential connected in n8n
- Slack app with:
  - Bot token
  - Interactivity enabled
  - Permission to call `chat.update`
  - Access to the target Slack channel
- A Google Sheet for ticket logging

## Import Order

1. Import `workflows/Support Approval Button Handler.json` first.
2. Configure its credentials and webhook path.
3. Activate the handler workflow.
4. Copy the production webhook URL from the handler workflow.
5. Add that URL to your Slack app's Interactivity Request URL.
6. Import `workflows/Customer Support Triage Agent.json`.
7. Configure its credentials, Google Sheet, and Slack destination.
8. Activate the main triage workflow.

## Placeholder Values

These workflows were sanitized for public sharing. After importing them into n8n, replace the placeholders with values from your own n8n instance and connected services.

### n8n Export Metadata

These values are export-specific and can usually be left as-is or regenerated by n8n after import:

| Placeholder | Meaning |
| --- | --- |
| `WORKFLOW_ID_PLACEHOLDER` | Original n8n workflow ID from the exported workflow. |
| `WORKFLOW_VERSION_ID_PLACEHOLDER` | Original n8n workflow version ID. |
| `N8N_INSTANCE_ID_PLACEHOLDER` | Original n8n instance ID. |

### Credentials

Reconnect these nodes to credentials in your own n8n instance:

| Placeholder | Replace With |
| --- | --- |
| `GMAIL_OAUTH2_CREDENTIAL_ID_PLACEHOLDER` | Your n8n Gmail OAuth2 credential ID. |
| `GMAIL_OAUTH2_CREDENTIAL_NAME_PLACEHOLDER` | Your Gmail credential name in n8n. |
| `GOOGLE_SHEETS_OAUTH2_CREDENTIAL_ID_PLACEHOLDER` | Your n8n Google Sheets OAuth2 credential ID. |
| `GOOGLE_SHEETS_OAUTH2_CREDENTIAL_NAME_PLACEHOLDER` | Your Google Sheets credential name in n8n. |
| `OPENAI_API_CREDENTIAL_ID_PLACEHOLDER` | Your n8n OpenAI API credential ID. |
| `OPENAI_API_CREDENTIAL_NAME_PLACEHOLDER` | Your OpenAI credential name in n8n. |
| `HTTP_HEADER_AUTH_CREDENTIAL_ID_PLACEHOLDER` | Your n8n HTTP Header Auth credential ID for Slack. |
| `HTTP_HEADER_AUTH_CREDENTIAL_NAME_PLACEHOLDER` | Your Slack HTTP Header Auth credential name in n8n. |

In n8n, the easiest way to replace these is usually to open each affected node and select your credential from the credential dropdown instead of editing JSON manually.

### Google Sheets

| Placeholder | Replace With |
| --- | --- |

| `YOUR_GOOGLE_SHEET_NAME` | The display name of your Google Sheet. |
| `YOUR_SHEET_TAB_ID` | The numeric gid of the sheet tab. |

### Slack

| Placeholder | Replace With |
| --- | --- |
| `YOUR_SLACK_CHANNEL_ID` | The Slack channel ID where review messages should be posted. |
| `YOUR_SLACK_WEBHOOK_OR_API_ENDPOINT` | Your Slack webhook/API endpoint if using the HTTP Request node as a webhook sender. |
| `YOUR_SLACK_INTERACTIVITY_WEBHOOK_PATH` | The webhook path used by the Slack approval handler workflow. |

The approval handler uses Slack's `chat.update` API endpoint:

```text
https://slack.com/api/chat.update
```

That public API URL does not need to be changed, but the HTTP Header Auth credential must contain your Slack bot token.

## Google Sheet Structure

Create a Google Sheet with columns that match the fields written by the workflows. The template expects ticket-tracking fields such as:

- `ticket_id`
- `customer_email`
- `subject`
- `body_text`
- `gmail_message_id`
- `gmail_thread_id`
- `conversation_type`
- `status`
- `ai_category`
- `ai_priority`
- `ai_sentiment`
- `ai_confidence`
- `ai_summary`
- `ai_customer_intent`
- `ai_suggested_reply`
- `final_reply`
- `slack_channel`
- `slack_message_ts`
- `approved_by`
- `approved_at`

If your imported Google Sheets nodes show missing columns, refresh the schema in each Google Sheets node after selecting your own spreadsheet and tab.

## Slack App Setup

1. Create or open your Slack app.
2. Add a bot token with permission to post and update messages.
3. Add the bot to your target support channel.
4. In n8n, create an HTTP Header Auth credential for Slack:

```text
Authorization: Bearer xoxb-your-slack-bot-token
```

5. In Slack app settings, enable Interactivity.
6. Set the Interactivity Request URL to the production webhook URL from `Support Approval Button Handler`.

## Gmail Setup

1. Create a Gmail label named `Support`, or update the Gmail Trigger node to use your preferred label.
2. Apply that label to incoming customer support emails.
3. Connect the Gmail Trigger and Gmail Reply nodes to your Gmail OAuth2 credential.
4. Test with a non-sensitive email before activating the workflow for production.

## OpenAI Setup

1. Add your OpenAI API credential in n8n.
2. Open the `AI Classify and Draft Reply` node.
3. Select your credential.
4. Review the configured model and prompt behavior.
5. Test the output against a sample support request before using it with real customers.

## Security Notes

- Do not commit real n8n credential IDs, Slack bot tokens, webhook URLs, customer emails, or production Google Sheet IDs.
- Keep Slack bot tokens in n8n credentials, not inside node JSON.
- Review imported workflows before activation.
- Use test inboxes, test Slack channels, and test Google Sheets when adapting the template.
- If you export your modified workflow later, sanitize the export again before sharing it publicly.
